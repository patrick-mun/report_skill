Skill report-formatter — Démonstration des corrections Phase 2
Audit visuel · 2 juin 2026 · Corrections V12 & V13

Corrections Phase 2 — Avant / Après

Contexte

Les corrections de la Phase 2a ont ajouté des classes CSS utilitaires à assets/report.css pour permettre aux documents générés d'éviter les styles inline codés en dur. Ce document démontre le résultat visuel de chaque correction, côte à côte.

  • V12 — Grille de conclusion : remplacer style="display:grid;…" par class="pillar-grid"
  • V13 — Barres de composition : remplacer style="background:#hex" par class="compo-*" et class="legend-*"

V13 — Barres de composition et légende

Les barres de composition d'ascendance utilisaient 10 couleurs hexadécimales codées en dur (5 barres + 5 swatches de légende). Avec les classes utilitaires, les couleurs sont liées aux variables CSS du projet.

❌ Avant — styles inline

Composition d'ascendance estimée :

Afrique / Malgache ≈ 45 % Inde du Sud ≈ 25 % Européen ≈ 15 % Zarabe / Gujarati ≈ 8 % Chinois / Asie ≈ 7 %

10 valeurs hex hardcodées — si la palette change, 10 attributs à modifier manuellement.

✅ Après — classes CSS du skill

Composition d'ascendance estimée :

Afrique / Malgache ≈ 45 % Inde du Sud ≈ 25 % Européen ≈ 15 % Zarabe / Gujarati ≈ 8 % Chinois / Asie ≈ 7 %

Couleurs via var(--green), var(--accent3), etc. — un seul endroit à modifier si la palette change.

Ce que ça change concrètement Le rendu visuel est identique. La différence est dans la maintenabilité : si le projet remplace --green: #10b981 par une autre valeur dans son override de palette, toutes les barres de composition se mettent à jour automatiquement, sans toucher au HTML.
