# Pagination — A4 layout with CSS @page + Paged.js

A4 pagination uses **CSS `@page` rules** plus the **Paged.js** polyfill. Content
is authored as a natural linear flow (semantic HTML), and pagination — page
breaks, a repeating footer, and `Page n / total` numbering — is produced
automatically. No rigid page boxes, no manual page management.

## Why Paged.js

CSS `@page` page breaks (`break-before`, `break-inside`) work in every browser's
print path. But **per-page footers and page numbers** rely on `@page`
*margin-boxes* (`@bottom-right { content: counter(page) }`), which **Chrome does
not support** in native print. Without help, a single `<footer>` in the flow
prints only once, and `counter(page)` never resolves — exactly the "no footer,
no page numbers" problem.

**Paged.js** is a small polyfill that implements the full `@page` model
(margin-boxes, `counter(page)`/`counter(pages)`, named pages, running elements).
It paginates the document in the browser — on screen *and* in print — so the
output stays a single self-contained HTML file printed with plain `Ctrl/Cmd + P`.

## How footers and page numbers work

Three pieces in `assets/report-linear.css` (already wired up — you don't write
these yourself, you just inline the CSS):

1. **Running footer.** The HTML `<footer class="page-footer">` is declared
   `position: running(docfooter)`, which lifts it out of the flow and lets it be
   repeated in a margin-box:
   ```css
   @page { @bottom-left { content: element(docfooter); } }
   ```
   Whatever text you put in that `<footer>` (short title · affiliation) appears
   at the bottom-left of **every** content page.

2. **Page numbers.** Generated by CSS, resolved by Paged.js:
   ```css
   @page { @bottom-right { content: "Page " counter(page) " / " counter(pages); } }
   ```
   `counter(pages)` is the real total — no estimation.

3. **Cover & TOC excluded.** The first page (cover) and the TOC carry no footer:
   ```css
   @page :first { @bottom-left { content: none; } @bottom-right { content: none; } }
   nav.toc-page { page: tocpage; }
   @page tocpage { @bottom-left { content: none; } @bottom-right { content: none; } }
   ```
   So the first *visible* `Page n / total` is on the first content page.

**On screen:** Paged.js renders the document as discrete A4 pages (you can see
the pagination, not just a continuous scroll).
**In print:** the same pages, with footers and numbers, export straight to PDF.

## Skeleton

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="utf-8" />
  <title>Report Title</title>
  <style>
    /* Inline assets/report-linear.css here (includes the @page rules above) */
  </style>

  <!-- Inline assets/paginate.js: sets window.PagedConfig (auto-paginate +
       accurate TOC page numbers). MUST come before the polyfill. -->
  <script>/* assets/paginate.js */</script>
  <!-- Inline assets/vendor/paged.polyfill.min.js: the Paged.js engine. -->
  <script>/* paged.polyfill.min.js */</script>
</head>
<body>
  <!-- Cover (page 1, no footer) -->
  <section class="cover">
    <h1>Report Title</h1>
    <p>Author, date, version.</p>
  </section>

  <!-- Table of Contents (page 2, no footer) -->
  <nav class="toc-page">
    <h2>Table of Contents</h2>
    <ol>
      <li><a class="toc-entry" href="#s1">
        <span class="toc-title">1. Section One</span>
        <span class="toc-dots"></span>
        <span class="toc-page-num"></span>   <!-- filled by paginate.js -->
      </a></li>
    </ol>
  </nav>

  <!-- Content (each h2 starts a new page; footer + page number on every page) -->
  <main>
    <section id="s1">
      <h2>1. Section One</h2>
      <p>Content.</p>
    </section>
  </main>

  <!-- The footer text — written ONCE, repeated on every content page -->
  <footer class="page-footer">
    <span>Report short title · Affiliation</span>
  </footer>
</body>
</html>
```

In practice you don't assemble this by hand: start from a file in `templates/`
and run the build (below), which inlines the CSS, the config, and the polyfill.

## Building a self-contained file

`build-inline-css.sh` turns every template/example into a single portable file:

```bash
cd report_skill
bash build-inline-css.sh
```

It inlines `report-linear.css`, `paginate.js`, and `paged.polyfill.min.js`,
normalizes the footer (the page number comes from CSS, so no `pageno` span), and
guarantees one `<style>`, one `</head>`, one `<body>`. A finished report is
~528 KB on disk (~105 KB gzipped) — the bulk is the inlined Paged.js engine,
the cost of being a true single file with reliable print pagination.

## Page breaks (authoring)

```css
h2 { break-before: page; }                 /* major sections start a new page */
table, figure, aside.callout { break-inside: avoid; }   /* never split a block */
```

To add a section, no renumbering is needed — add the block and a TOC entry:

```html
<section id="s5"><h2>5. New Section</h2><p>…</p></section>
<!-- in the TOC -->
<li><a class="toc-entry" href="#s5">
  <span class="toc-title">5. New Section</span>
  <span class="toc-dots"></span><span class="toc-page-num"></span>
</a></li>
```

The `h2` triggers the page break; `paginate.js` fills the page number after
Paged.js lays the document out.

## Best practices for readable pages

- **One major topic per `h2`.** Each `<h2>` starts a new page — align section
  boundaries with topic boundaries.
- **`<h3>` for subsections** (no page break).
- **Tables/figures whole.** `break-inside: avoid` keeps them together; if a table
  can't fit with surrounding text, give it its own page.
- **Usable height ≈ 257 mm** per page (297 − 20 − 20 mm margins): ~40–50 lines of
  body text, fewer with large visuals.
- **Color in print:** `report-linear.css` sets `print-color-adjust: exact` on
  visual components; still enable **Background graphics** in the print dialog.

## Print checklist (before PDF export)

1. **Open the HTML in Chrome or Firefox.** Paged.js paginates on load — you'll
   see discrete A4 pages, with footers and `Page n / total` on content pages.
2. **Scan the pages:** `h2` headings start sections (not orphaned), tables/figures
   aren't split, cover and TOC have no footer, content pages are numbered.
3. **Print preview (`Ctrl/Cmd + P`):**
   - **Margins:** *Default* — the `@page` rule already supplies the 20 mm margin
     (do **not** set "None", or the footer margin-boxes get clipped).
   - **Background graphics:** *On* (so stat cards, callouts, bars print in color).
   - **Paper size:** A4. **Orientation:** Portrait.
4. **Save as PDF**, then spot-check a few pages: footer text + page number present
   on every content page, colors preserved, breaks clean.

### Troubleshooting

- **No footer / no page numbers:** confirm both scripts are present and in order —
  `paginate.js` (which sets `window.PagedConfig`) **before**
  `paged.polyfill.min.js`. If the polyfill is missing, the footer falls back to
  rendering once in flow and numbers won't resolve.
- **TOC numbers blank:** Paged.js must finish before they fill; they populate
  automatically via `paginate.js`. Ensure each TOC link's `href="#sX"` matches a
  section `id="sX"`.
- **Footer clipped in PDF:** margins were set to "None" — use "Default".

## Updating Paged.js

The engine is vendored at `assets/vendor/paged.polyfill.min.js` (Paged.js
v0.4.3). To update, replace that file from the npm package `pagedjs`
(`dist/paged.polyfill.min.js`) and re-run `build-inline-css.sh`.
